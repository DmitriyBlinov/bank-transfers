Тестовое задание Java Developer

Требования
Стек: Java SE 8+, система логирования на основе готового решения, библиотеки и фреймворки (по желанию)

Логирование должно осуществляться при любом действии, которое вызывает изменение данных. Также при разработке должны быть учтены возникающие ошибки (их также требуется корректно логировать)

Структура данных
Сущность Аккаунта с полями: id (string), money (int)

После запуска нужно создать 4 (или более) экземпляров сущности Аккаунт (id должен быть случайным) и 10000 на счете. После этого приложение запускает не менее двух независимых потоков. Каждый поток просыпается раз в 1000 - 2000 мс - число из этого интервала при каждой инициализации генерируется случайным образом.

Каждый из потоков выполняет перевод суммы между счетами - суммы также генерируются случайным образом. Поле счета всегда содержит положительное значение, общая сумма всех аккаунтов остается одинаковой после всех операций (максимальное число операций - 30).

Предусмотреть количество счетов и потоков, а также параллельный перевод средств между счетами. Результат операций требуется логировать в файл.
